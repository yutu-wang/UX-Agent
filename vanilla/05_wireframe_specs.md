# T-Pass 線框稿規格（Wireframe Specs）

每個畫面說明：用途、版面區塊、元件清單、互動行為、邊緣狀態。

## Screen 01：歡迎頁（Splash / Onboarding Welcome）

**用途**：App 首次開啟，引導用戶登入或申辦

**版面區塊**
```
┌─────────────────────┐
│   [Logo / 品牌視覺]  │  區塊 A：品牌識別
│                     │
│  「全台通用，每月省錢」│  區塊 B：主標題
│  「通勤、上學都適用」 │         副標題
│                     │
│  [立即申辦]  ← CTA  │  區塊 C：主要行動
│  [已有帳號，登入]    │         次要行動
└─────────────────────┘
```

**元件**：Logo、H1 標題、Body 文字、Primary Button、Text Link
**邊緣狀態**：已登入用戶直接跳至首頁，不顯示此頁

## 彈窗 / Bottom Sheet 清單

| # | 名稱 | 觸發點 | 內容 |
|---|------|--------|------|
| 1 | 刪除付款方式確認 | 點擊刪除按鈕 | 警示文字 + 確認 / 取消 |
| 2 | 取消方案確認 | 點擊取消方案 | 說明生效日期 + 確認 / 取消 |
| 3 | 路線詳情 | 點擊路線列表項目 | 路線圖、適用票價 |
| 4 | 單筆紀錄詳情 | 點擊紀錄列表項目 | 時間、路線、原價、折抵金額 |
| 5 | 推播通知 | 首次使用刷卡成功 | 「T-Pass 已生效，本次折抵 NT$35」 |
| 6 | 審核結果通知 | 學生審核完成 | 通過 / 未通過 + 下一步指引 |

## 錯誤狀態畫面

| # | 情境 | 畫面內容 |
|---|------|---------|
| E1 | 網路連線失敗 | 圖示 + 「無法連線，請確認網路」+ 重試按鈕 |
| E2 | 刷卡 / 扣款失敗 | 圖示 + 失敗原因 + 「更換付款方式」按鈕 |
| E3 | 學生審核未通過 | 原因說明 + 「重新上傳」按鈕 |
