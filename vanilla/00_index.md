# T-Pass Prototype 設計素材索引

## 檔案清單

| 檔案 | 內容 | 用途 |
|------|------|------|
| `01_personas.md` | 4 位使用者人物誌 | 設計決策依據，招募測試受測者 |
| `02_journey_map.md` | 旅程地圖（通勤族主線 + 學生差異） | 找出設計機會點 |
| `03_information_architecture.md` | App Sitemap、頁面數量統計 | Figma 檔案結構規劃 |
| `04_user_flows.md` | 6 個核心流程（含優先級） | Prototype 互動連線依據 |
| `05_wireframe_specs.md` | 9 個主畫面 + 6 個彈窗規格 | 線框稿繪製參考 |
| `06_content_inventory.md` | 所有文案、格式規範 | 填入 Figma 元件的實際文字 |
| `07_interaction_notes.md` | 動畫、手勢、無障礙規範 | 高擬真 Prototype 設定依據 |

## 建議 Prototype 製作順序

1. **建立元件庫**：參考 `06` 的文案與 `07` 的規範
2. **繪製主要畫面**：依 `05` 的規格，優先做 Screen 01–06
3. **連接核心流程**：依 `04` Flow 1（申辦）和 Flow 3（查看狀況）
4. **加入互動細節**：參考 `07` 設定轉場與動畫
5. **驗證易用性**：對照 `01` 人物誌招募測試受測者
