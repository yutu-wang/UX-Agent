# T-Pass 互動設計備註（Interaction Notes）

動畫、微互動、狀態轉換與無障礙規範。

---

## 頁面轉場

| 情境 | 動畫 | 時長 |
|------|------|------|
| Tab 切換 | Cross-fade | 200ms |
| Push（進入子頁） | Slide from right | 300ms |
| Modal / Bottom Sheet 展開 | Slide up | 250ms |
| Modal 關閉 | Slide down | 200ms |
| 申辦步驟前進 | Slide left | 250ms |
| 申辦步驟返回 | Slide right | 250ms |
| 成功頁進入 | Scale up + fade in | 400ms |

---

## 表單互動

**即時驗證規則**

| 欄位 | 驗證時機 | 錯誤訊息 |
|------|---------|---------|
| 手機號碼 | onBlur | 請輸入有效的台灣手機號碼 |
| 驗證碼 | onChange（6碼滿後自動送出） | 驗證碼錯誤，請重新輸入 |
| 信用卡號 | onChange（每4碼自動加空格） | 請輸入有效的卡號 |
| 到期日 | onBlur | 卡片已過期 |
| 費用試算欄位 | onBlur | 請輸入有效的次數（1–200） |

**按鈕狀態**
- Default → Hover（桌面）→ Pressed → Loading（spinner）→ Success / Error
- 必填欄位未完成：按鈕 `disabled`（降低 opacity 至 40%）
- Loading 狀態：按鈕文字替換為旋轉圓圈，防止重複點擊

## 核心元件互動

### 主視覺卡片（首頁省錢數字）
- 進入首頁時：數字從 0 向上計數至實際金額（300ms，easeOut）
- 點擊卡片：展開本月詳細統計（Bottom Sheet）

### 月份切換器（紀錄頁）
- 左右箭頭：Tap → 滑動至相鄰月份，資料動態載入
- 未來月份：箭頭 disabled，不可點擊
- 載入中：列表顯示 Skeleton Loader（3 筆佔位列）

### Radio Card（方案選擇）
- 未選中：白底、灰框（1px）
- 選中：品牌色框（2px）+ 左側勾選圖示 + 淡色背景
- Tap 動效：Scale 0.98 → 1.0，100ms

### Bottom Sheet
- 背景遮罩：黑色 40% opacity，Tap 可關閉
- 拖曳把手：頂部顯示 4×28px 灰色橫條
- 支援上滑展開 / 下滑關閉手勢

## 載入狀態

| 情境 | 處理方式 |
|------|---------|
| 首頁資料載入 | Skeleton Card（灰色佔位塊） |
| 紀錄列表載入 | Skeleton 列（3 筆） |
| 申辦送出中 | 全頁 Overlay + 旋轉圖示 + 「申辦中，請稍候」 |
| 圖片上傳中（學生證） | 進度條 0→100% |
| 付款授權中 | 全頁 Overlay + 文字「連線至付款服務中」 |

---

## 手勢支援

| 手勢 | 觸發區域 | 行為 |
|------|---------|------|
| 下拉刷新 | 首頁、紀錄頁列表 | 重新載入資料 |
| 左滑 | 付款方式列表項目 | 顯示「刪除」按鈕 |
| 上滑關閉 | Bottom Sheet | 關閉底部彈窗 |
| 返回手勢（iOS） | 全域 | 返回上一頁 |

## 無障礙規範（Accessibility）

| 項目 | 規範 |
|------|------|
| 對比度 | 文字與背景對比 ≥ 4.5:1（WCAG AA） |
| 最小點擊區域 | 44×44pt |
| VoiceOver / TalkBack | 所有互動元素需有 `accessibilityLabel` |
| 動態字體 | 支援 iOS Dynamic Type，最大字級仍需可讀 |
| 錯誤提示 | 不得只靠顏色傳達，需同時顯示文字 |
| 圖示 | 需搭配文字標籤（尤其底部 Tab） |

---

## 空狀態設計原則

1. **說明原因**：告訴使用者為何沒有資料
2. **給予行動**：提供一個主要 CTA 按鈕
3. **插圖風格**：輕量線條插圖，避免過度設計
4. **禁止使用「暫無資料」**：過於技術性，改用具情境的說明文字
