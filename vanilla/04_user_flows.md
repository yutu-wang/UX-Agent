# T-Pass 使用者流程（User Flows）

---

## Flow 1：首次申辦（一般使用者）

```
[啟動 App]
     │
     ▼
[歡迎頁] → [登入 / 註冊]
                │
                ▼
         [手機號碼輸入]
                │
                ▼
         [簡訊驗證碼]
                │
         ┌──── ▼ ────┐
         │ 驗證成功   │ 驗證失敗
         │            └──→ [重新發送] → 回到驗證碼輸入
         ▼
   [設定密碼]
         │
         ▼
   [選擇縣市 & 方案]
         │
         ▼
   [確認適用資格] → 一般用戶 → 繼續
         │
         ▼
   [綁定付款方式]
         │
    ┌────┴────┐
    │ 成功    │ 失敗（卡片無效）
    │         └──→ [錯誤提示] → 重新輸入
    ▼
[申辦完成頁]（顯示：方案名稱、生效日期、使用說明）
         │
         ▼
   [進入首頁]
```

---

## Flow 2：首次申辦（學生優惠）

```
[確認適用資格] → 選擇「學生」
         │
         ▼
   [上傳學生證]
         │
         ▼
   [送出審核] → [等待審核頁]（預計 1–3 個工作天）
         │
    ┌────┴────┐
    │ 審核通過 │ 審核不通過
    │  (推播通知)│  (說明原因 + 重新上傳)
    ▼
[綁定付款方式] → 同 Flow 1 後續
```

---

## Flow 3：查看本月使用狀況

```
[首頁]
  │
  ├── [本月省下金額卡片] → 點擊 → [本月詳細統計頁]
  │
  └── [搭乘紀錄 Tab]
            │
            ▼
      [月份列表（預設本月）]
            │
            ▼
      [單筆紀錄展開] → 顯示：路線、時間、原價、折抵後金額
```

---

## Flow 4：更換付款方式

```
[帳戶 Tab]
     │
     ▼
[付款方式]
     │
     ▼
[已綁定卡片列表]
     │
  ┌──┴──┐
  │ 新增 │ 刪除
  │      └──→ [確認刪除彈窗] → 確認 → 刪除成功 Toast
  ▼
[選擇付款類型]（信用卡 / 金融卡 / 行動支付）
     │
     ▼
[輸入卡片資訊] 或 [跳轉行動支付授權]
     │
  ┌──┴──┐
  │ 成功 │ 失敗
  │      └──→ [錯誤訊息] → 重試 or 返回
  ▼
[設為預設付款方式？] → 是 / 否
     │
     ▼
[新增成功 Toast] → 返回付款方式列表
```

---

## Flow 5：查詢適用路線

```
[方案 Tab] → [適用路線查詢]
     │
     ▼
[縣市篩選]（北北基、桃園、台中、高雄…）
     │
     ▼
[路線列表]（捷運、公車分類）
     │
     ▼
[單一路線詳情] → 顯示：路線名稱、起訖站、適用票價說明
```

---

## Flow 6：問題回報

```
[帳戶 Tab] → [客服 / 問題回報]
     │
  ┌──┴────────┐
  │ FAQ        │ 線上回報
  │            ▼
  │     [選擇問題類型]
  │       ├── 折扣未套用
  │       ├── 刷卡失敗
  │       ├── 帳務問題
  │       └── 其他
  │            │
  │            ▼
  │     [填寫說明 + 上傳截圖（選填）]
  │            │
  │            ▼
  │     [送出] → [回報成功頁]（顯示案件編號）
  │
  └── [常見問題列表] → 搜尋 or 分類瀏覽
```

---

## 流程優先級

| 流程 | 原型重要性 | 說明 |
|------|-----------|------|
| Flow 1 申辦 | 🔴 最高 | 轉換關鍵，直接影響申辦率 |
| Flow 3 查看狀況 | 🔴 最高 | 日常核心需求 |
| Flow 4 更換付款 | 🟡 中 | 高頻痛點 |
| Flow 2 學生申辦 | 🟡 中 | 特定族群但需求明確 |
| Flow 5 查路線 | 🟢 低 | 輔助功能 |
| Flow 6 問題回報 | 🟢 低 | 補救性流程 |
